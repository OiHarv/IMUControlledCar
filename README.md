# IMUControlledCar
An IMU Controlled RC Car for [Hack Club Fallout](https://fallout.hackclub.com)


This project is a small RC Car with an attachment at the top meant for a weapon. 
Instead of being controlled with a normal joystick remote, though, it is controlled via hand movement through an IMU (Inertial Measurement Unit) inside the remote, which in turn is strapped to your hand to make controlling the robot feel more natural.

The way you use it is simply by putting the remote on your hand, starting up the firmware and calibrating the IMU if needed, turning the car on and controlling it right then and there.
Holding your hand out, with your palm parallel to the floor will have the car not move at all. Turning your palm so it faces away from you makes it go backwards and tilting it towards you makes it go forwards. Tilting it to the left or right makes it move in the those directions.
Any questions on how to build it can be answered in the [Assembly Guide](AssemblyGuide.md) (Seperate document. Step-by-Step instructions on exactly what to do, in order)

Even before [Hack Club Fallout](https://fallout.hackclub.com), I wanted to make this project. My younger siblings have many remote controlled cars and I noticed that some of the controls were harder for the youngest to use / grasp. I thought: "what's more intuitive than using your hand?" and thought it would be cool to make this project. I had no idea if it had been done before, but I figured it would be cool and would teach me a lot.

The original idea was to have it read if you brought your hand up, it would trigger the car to jump up, but I felt that it would be more fun if it controlled a weapon attachment. Then my younger siblings could battle against each other. This also has the added bonus of making the code much simpler.

This project is meant to be edited. If you want to change the face from being a pirate to an astronaut, be my guest. If you want to add the jump feature or a twirl feature, the code is open and everything is downloadable. All questions about the assembly and any files needed should already be in this repository. If not, just comment the question.

Any questions on how to build it can be found in the [Assembly Guide](AssemblyGuide.md) (Seperate document. Step-by-Step instructions on how to build it start to finish). Any files, images, etc. are in properly marked folders elsewhere in the project. This document is meant to explain what and why this project is.
